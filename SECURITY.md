# Security Policy

## Reporting a vulnerability

Do not disclose credentials, private papers, generated research artifacts, or vulnerability details in a public issue.

Use GitHub Security Advisories for private reports when available. Otherwise, contact the repository owner through the GitHub profile and share only enough information to establish a private reporting channel.

If an API key may have been exposed, revoke it immediately and issue a replacement.

## Sensitive research data

Local configuration, papers, caches, research memory, and generated outputs are excluded from version control. Contributors must confirm that fixtures and example documents may be redistributed before committing them.
